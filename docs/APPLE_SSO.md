# Sign in with Apple setup

Required for App Store approval if you also offer Google sign-in (Apple Guideline 4.8). Plumbing is identical to [Google SSO](GOOGLE_SSO.md) — Apple is just another OIDC provider in Ory.

Plan for ~30 min the first time. Most of it is Apple Developer Portal clicks.

---

## 1. Configure the App ID

You already created `cc.lovelinuxalot.familygram` at https://developer.apple.com/account/resources/identifiers/list during the iOS setup. Now enable the capability.

1. Open the identifier → click **Edit**.
2. Scroll to **Capabilities** → check **Sign In with Apple**.
3. **Save** → confirm.

---

## 2. Create a Services ID

Apple distinguishes the **App ID** (your iOS app's bundle identifier) from the **Services ID** (the OAuth client for "Sign in with Apple" on the web). Both are needed.

1. https://developer.apple.com/account/resources/identifiers/list → **+** → **Services IDs** → **Continue**.
2. **Description**: `Familygram Sign in with Apple` (any human-readable label).
3. **Identifier**: `cc.lovelinuxalot.familygram.signin` (must be unique, distinct from the App ID).
4. **Continue → Register**.
5. Click the new Services ID → check **Sign In with Apple** → **Configure**:
   - **Primary App ID**: `cc.lovelinuxalot.familygram`.
   - **Domains and Subdomains**: paste your Ory project's domain — `<your-slug>.projects.oryapis.com` (no `https://`, no path).
   - **Return URLs**: leave blank for now — Ory will print one in step 4.
   - **Save** → **Continue** → **Save**.

Note the **Services ID** (`cc.lovelinuxalot.familygram.signin`) and your **Team ID** (top right of the Developer Portal, 10 chars).

---

## 3. Create a Sign in with Apple key

This is the private key Apple uses to issue identity tokens. Ory needs it.

1. https://developer.apple.com/account/resources/authkeys/list → **+**.
2. **Key Name**: `Familygram SIWA`.
3. Check **Sign In with Apple** → **Configure** → primary App ID `cc.lovelinuxalot.familygram` → **Save**.
4. **Continue** → **Register** → **Download**. Apple shows the `.p8` file **only once**. Save it.
5. Note the **Key ID** (10 chars, shown next to the key name).

---

## 4. Add Apple as a social provider in Ory

1. Ory Console → **Authentication → Social Sign-In (OIDC)** → **Add provider** → **Apple**.
2. Fill in. The Ory UI uses generic OIDC field names ("Client ID") rather than Apple's terminology — the mapping:

   | Ory field                                    | What to paste                              |
   |----------------------------------------------|--------------------------------------------|
   | **Client ID**                                | The **Services ID** from step 2 — e.g. `cc.lovelinuxalot.familygram.signin`. **NOT** the App ID (`cc.lovelinuxalot.familygram`). |
   | **Apple Team ID**                            | Your 10-char Team ID — visible top-right in https://developer.apple.com/account. |
   | **Apple Private Key ID**                     | Key ID from step 3 (10 chars).             |
   | **Apple Private Key**                        | Contents of the `.p8` file, including the `-----BEGIN PRIVATE KEY-----` and `-----END PRIVATE KEY-----` lines. |

   The "Client Secret" field that Ory shows for other OIDC providers does **not** apply to Apple — Apple uses a JWT signed with your `.p8` as the secret, which Ory generates internally from the Team ID + Key ID + private key you supplied above. Leave Client Secret blank if it's shown.
3. Expand **Advanced settings**:
   - **Scope**: `name`, `email` (two entries).
   - **Data mapping (Jsonnet)**:

     ```jsonnet
     local claims = std.extVar('claims');
     {
       identity: {
         traits: {
           email: claims.email,
           // Apple sends "name" as { firstName, lastName } in the first sign-in
           // only. Defensive defaults in case fields are missing on returning users.
           name: {
             first: if 'given_name' in claims then claims.given_name else '',
             last:  if 'family_name' in claims then claims.family_name else '',
           },
           // Apple doesn't supply a profile picture.
         },
       },
     }
     ```
4. Save. Ory shows the **Authorized Redirect URI** for Apple on the provider page — looks like:
   ```
   https://<your-slug>.projects.oryapis.com/self-service/methods/oidc/callback/apple-XXXXX
   ```
   The suffix (`apple-XXXXX`) is generated by Ory and is unique per provider config. **Copy the URL exactly, including the suffix** — Apple's Return URL validation is a literal string match. If you've already pasted a different URL (e.g. you guessed the path) update it.

   If you can't see the URL in the Ory Console after saving, you can also derive it by triggering a sign-in from the app — when Apple errors with "Invalid web redirect url", the failing URL is in the browser address bar. Copy from there.
5. Back in Apple's **Services ID** (step 2.5), **Configure** → **Return URLs** → paste the URL → **Save**.

`familygram://callback` (the project-level allowed return URL) is already configured from the Google setup — no extra step.

---

## 5. Test in the app

1. Add your Apple ID's email to the Familygram allowlist (Admin panel) or to `ADMIN_EMAILS`.
2. Rebuild: `make clean && make setup && make dev`.
3. Login screen → tap **Continue with Apple**.
4. Native iOS Apple sign-in sheet (Face ID prompt + email choice).
5. **Important — do not pick "Hide my email"**. Familygram's allowlist matches the email exactly, and Apple's relay email (`xxx@privaterelay.appleid.com`) won't be in your admin allowlist. Choose **"Share my email"**.
6. Done — lands on the feed.

---

## "Hide my email" — why we don't support it (yet)

When a user picks "Hide my email", Apple gives our app a per-user relay email like `abc123@privaterelay.appleid.com`. Familygram's allowlist requires admins to pre-allowlist real emails, which they don't know in advance. There are two ways to fix this later:

- **Stop using email as the allowlist key** — switch to a "join code" the admin generates per-invite. Bigger change.
- **Allow post-hoc admin approval** — first-time users land on a "waiting for admin" screen; admin approves from the panel.

For now: just instruct family members to choose "Share my email" when signing in with Apple. They can change later in iOS Settings → Apple ID → Sign in with Apple if they regret it.

## Troubleshooting

**"No Apple provider found in the Ory login flow"** — provider isn't enabled in Ory Console (step 4) or the project URL the app uses doesn't match. Verify `ORY_BASE` in `scripts/ship.env` matches your Ory project.

**Apple shows "invalid_client" in the browser** — Services ID, Team ID, Key ID, or .p8 in Ory is wrong. Re-check step 4 fields.

**Apple consent screen but never returns to app** — `familygram://callback` isn't in Ory's allowed return URLs. Re-do step 4 of [GOOGLE_SSO.md](GOOGLE_SSO.md) (the project-level Browser redirects). It's shared across all OIDC providers.

**"Your email isn't on the family list" after Apple sign-in** — user picked "Hide my email" or their Apple ID email isn't in the allowlist. Have them sign in again with "Share my email," or add their real email to the allowlist.
